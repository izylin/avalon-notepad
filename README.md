# 阿瓦隆笔记本（Avalon Notepad）

配合线下阿瓦隆桌游使用的移动端/网页记录工具。用于记录组队、投票与任务结果，全部数据只保存在本机浏览器（`localStorage`），不上传、不联网存储。

## 产品定位

一款配合线下阿瓦隆游戏使用的记录工具，帮助玩家在线下对局中记录身份标签、投票及任务结果，支持中途保存进度与赛后复盘查看。

## 核心流程

首页 → 新开一局（选人数、选角色、选首轮队长）→ 生成座位圆桌 →
队长选人组队 → 全员表决（组队投票）→
- 若投票通过 → 上车执行任务 → 记录任务成功/失败 → 累计至一方达成 3 胜或触发刺杀 → 游戏结束
- 若投票未通过 → 否决计数 +1，更换队长重新组队；连续 **5 次**组队被否决则直接判**红方（坏人阵营）获胜**（流车规则）

游戏结束后展示胜负结果与任务回顾，可返回首页；进行中的对局可随时保存到本机存档，下次「从存档继续」。

## 已实现功能（MVP）

- **新开一局**：选择 5–10 人局，勾选本局特殊角色（梅林 / 派西维尔 / 莫甘娜 / 刺客 / 莫德雷德 / 奥伯伦），自动按官方配置补齐忠臣/爪牙人数，选择首轮队长
- **圆桌座位图**：按人数顺时针生成座位，实时高亮队长、当前组队成员
- **组队 → 投票 → 任务**逐轮记录：
  - 队长选人组队
  - 记录全员赞成/反对票，自动判定通过/否决
  - 连续否决计数与达到 5 次的流车判定
  - 记录任务执行的失败票数量，自动判定任务成功/失败（8 人局第 4 局需 2 张失败票）
- **身份标签**：可在对局中标记/取消标记座位（如「派西维尔识别的梅林候选」），按任务节点生效，可回看历史任务时的标签状态
- **发车记录**：按任务回看每一次组队的队长、队伍、投票明细与结果
- **刺杀环节**：蓝方达成 3 胜且场上有刺客时，记录刺杀成功/失败，决定最终胜负
- **笔记**：自由文本记录身份推测、发言与复盘，随对局自动保存
- **本机存档**：对局状态实时写入 `localStorage`，支持中途退出后「从存档继续」
- **规则介绍**：阵营目标、组队投票、任务结算规则说明

## 当前实现与 PRD 的已知差异

- **玩家昵称**：PRD 设想录入全部玩家昵称，当前版本以座位编号（我 / 2号 / 3号…）代替，尚未支持自定义昵称
- **远征投票 vs 组队投票**：当前仅记录一轮「组队投票」（赞成/反对上车），未区分 PRD 提到的「远征投票」与「组队投票」两类

## 技术栈

- [Next.js](https://nextjs.org/) 16（App Router）+ React 19 + TypeScript
- 纯 CSS（无 UI 框架），状态与游戏规则集中在 [`src/lib/game.ts`](src/lib/game.ts)
- 无后端，数据只存于浏览器 `localStorage`

## 目录结构

```
src/
  app/
    page.tsx        # 页面路由与各屏幕（首页/新开一局/对局/笔记/刺杀/结算）
    layout.tsx       # 根布局
    globals.css       # 全局样式（阿瓦隆主题）
  components/         # SeatSvg、MissionPager、LaunchHistory 等 UI 组件
  lib/
    game.ts           # 游戏状态、角色配置、任务规则与结算逻辑
```
